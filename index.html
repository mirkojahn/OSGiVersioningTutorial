OSGiVersioningTutorial by mirkojahn

A tutorial project explaining how osgi versioning works

Versioning in OSGi is a sophisticated task. In order to help and educate people on how versioning can be done, this project was created. The project resembles various steps in a typical evolution of a (very simple) API. Starting from a rather naive approach, going through slight enhancements to API breaking changes. While doing so, versions are applied in a way, that changes can be track accurately and as far as possible, existing code can be reused and multiple versions of the same software can be deployed and used in the same runtime at the same time. A tutorial (presentation or wiki - not done yet) will explain all steps in detail and point out the limitations of the current state of the art. Enhancements, contributions, bug reports, suggestions - in short: everything that helps to improve are very welcome.

Dependencies

Git (optional for source checkout) Java 5 or higher Eclipse (3.5 - lower versions will probably work as well, but are not tested)

Install

Download the latest version from the repository. Each folder in the root represent a unique Eclipse project that should be imported into an Eclipse Workspace. After having done that, you'll find various launch configurations in the "Run" menu. Pick the one you're interested in. All of these configurations can be found in the net.mjahn.osgi.tutorials project in case you're curious.
